# 题目 1：Prefix 吞吐

## 目标

在固定 benchmark 脚本和固定数据集的前提下，尽可能提高最终上报的吞吐指标。

## 评分指标

官方指标为 benchmark 脚本打印出来的 `prompt_tokens_per_sec`。

数值越高越好。

## 满分线

- 当 `prompt_tokens_per_sec >= 291000` 时，视为达到满分线。

## 固定条件

- 模型路径固定为 `/ceph/arknet/hf_models/Qwen/Qwen3-14B`
- 数据集文件固定
- prompt 顺序固定
- sampling 参数固定
- 必须使用提供的 Python API 工作流，也就是基于 `LLM(...)`
- 不允许把这道题改写成 HTTP serving 任务

## 限制条件

- 不允许下载任何模型、tokenizer、checkpoint、adapter 或额外权重
- 不允许使用网络访问作为解题手段
- 不允许修改数据集文件
- 不允许修改指标计算逻辑
- 不允许使用第二个模型
- 不允许使用外部缓存服务
- 不允许跨多台机器做分布式执行

## 允许事项

- 允许在正式计分前增加少量辅助步骤
- 但正式计分时使用的原始请求集合、顺序与数据集内容必须保持不变

- 你实际运行的完整命令
- 输出得到的 JSON 结果文件
- 一段简短说明，描述你做了哪些改动
